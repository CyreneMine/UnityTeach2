## 本次理解

`Mathf.Lerp(start, end, t)` 的第三个参数 `t` 不是自动意义上的“秒数”，而是从 `start` 到 `end` 的插值进度。

- `t = 0`：结果等于开始值；
- `t = 1`：结果等于终点值；
- `t = 0.5`：结果在开始值和终点值的中间；
- 如果让 `t` 用 `Time.deltaTime` 从 `0` 累加到 `1`，它才表现为大约 1 秒完成一次从起点到终点的移动。

可以把它理解成：

```csharp
result = start + (end - start) * t;
```

## 第一种写法：曲线跟随

第一种写法类似：

```csharp
current = Mathf.Lerp(current, target, Time.deltaTime);
```

这里每一帧都把当前值重新当成新的开始值，所以它不是“走完整段路的固定进度”，而是“每帧走剩余距离的一小部分”。

因此现象是：

- 一开始离目标远，剩余距离大，每帧变化量也大；
- 越靠近目标，剩余距离越小，每帧变化量也越小；
- 最后会无限接近目标，看起来几乎不动；
- 数学上很难真正等于目标点。

这就是先快后慢的原因。它更像缓动追踪，不是标准匀速移动。

## 第二种写法：固定起点和目标点

第二种写法的核心是：

```csharp
time += Time.deltaTime;
pos = Mathf.Lerp(startPosition, targetPosition, time);
```

当检测到目标位置变化时：

- 记录新的目标位置；
- 记录当前方块位置作为本次移动的开始位置；
- 把 `time` 重置为 `0`；
- 然后让 `time` 逐帧累加到 `1`。

这样 `startPosition` 和 `targetPosition` 在一次移动周期内保持不变，`time` 才是真正从 `0` 走到 `1` 的进度。

如果目标物体移动是一次性的，之后不再改变位置，那么当 `time >= 1` 时，方块就会到达目标点。

## 修正后的理解

`Lerp` 本身不会自动花 1 秒，它只根据第三个参数 `t` 返回一个插值结果。

当 `t` 是从 `0` 累加到 `1` 的值时，`Lerp` 才表现为从开始点到终点的完整移动过程。

第一种写法先快后慢，是因为每帧都把当前位置作为新的开始点，导致它一直在计算剩余距离的一部分。

第二种写法更适合本题，因为它固定一次移动的开始点和终点，再用累加出来的 `t` 表示本次移动进度。
